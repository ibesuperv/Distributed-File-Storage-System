# Contributing to Distributed-File-Storage-System

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 🌈 Our Philosophy
We aim for **Google-tier engineering quality**: performance, reliability, and clean, idiomatic Go.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Open an issue with a clear title and description.
- Include as much relevant information as possible, including steps to reproduce the bug.
- Mention your Go version and OS.

### Suggesting Enhancements
- Check the issues list to see if the enhancement has already been suggested.
- Open a new issue to discuss the enhancement before starting work on a Pull Request.

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **Write clean code**: Follow standard Go formatting (`go fmt`).
3. **Include tests**: If you're adding a feature, add unit tests in the relevant file.
4. **Document your changes**: Update `README.md` or the `docs/` folder if necessary.
5. **Issue the PR**: Describe exactly what you changed and why.

## 📝 Style Guide
- Use descriptive naming (e.g., `NewTCPPeer` instead of `NewP`).
- Keep functions small and focused on a single responsibility.
- Document exported functions.
- Handle all errors explicitly.
